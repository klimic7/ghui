# Codex Comment Implement

## Why

Review comments often describe small, actionable fixes. Today ghui can read, reply, edit, and delete comments, but acting on a comment requires leaving the TUI, manually prompting Codex, inspecting the patch, committing, pushing, replying, and resolving the thread.

## What we'd ship

- A Comments view shortcut (`i`) on a selected review comment: **Implement comment**.
- An in-app modal that runs Codex against the selected comment and current PR context.
- The modal shows Codex's result and the resulting local `git diff`.
- Confirming the modal commits and pushes the change, replies to the selected review comment with a short resolution note, and resolves the GitHub review thread.
- If the selected comment is not an inline review comment, or the current directory is not the PR's local checkout, ghui shows a clear error instead of attempting changes.

## API / Architecture Mapping

- UI:
  - Extend Comments view keymap with `i`.
  - Add `CommentImplementationModal` with states: running / ready / confirming / done / error.
  - Modal keymap: scroll, `enter` confirm, `esc` close.
- Services:
  - `CodexExplainer` or a new `CodexCommentImplementer` runs `codex exec` with workspace write access.
  - A git helper uses `git status --porcelain`, `git diff`, `git commit`, and `git push`.
  - `GitHubService` grows `resolveReviewThread(threadId)` via GraphQL.
  - `replyToReviewComment` already exists and can post the resolution reply.
- Data:
  - `PullRequestReviewComment` needs optional `threadId` so a selected comment can resolve its GitHub review thread.
  - GraphQL review-thread hydration already exists for missing replies; attach thread id during that merge path.

## Open Questions

- Whether ghui should auto-checkout the PR branch when the local checkout is elsewhere. Lean for v1: no, require the user to be in the right checkout.
- Whether the commit message should be generated by Codex or deterministic from the comment. Lean: deterministic first, e.g. `Address review comment`.
- Whether confirmation should allow editing the reply body or commit message. Lean: show generated defaults in v1, editable later.

## Out of Scope (v1)

- Running Codex against remote-only repositories without a local checkout.
- Auto-resolving issue comments, because only review threads have a GitHub resolve API.
- Interactive Codex chat inside ghui.
- Staging partial hunks; v1 commits the full working tree diff produced by the action after confirming.
